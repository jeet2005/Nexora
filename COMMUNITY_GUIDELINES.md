# Nexora Community Guidelines

## Purpose

Nexora is built for students, researchers, developers, testers, and early users who want to improve predictive analytics workflows together. The community feedback system exists to collect useful reports, evaluate ideas transparently, recognize contributors, and turn high-quality feedback into product improvements.

These guidelines define how users and administrators should participate in Nexora community spaces, including feedback submissions, public profiles, leaderboards, reactions, administrator replies, and contribution recognition.

## Community Standards

All contributors are expected to participate professionally and constructively.

- Be respectful when discussing bugs, limitations, or design decisions.
- Keep feedback specific, actionable, and relevant to Nexora.
- Use clear titles and descriptions so administrators can triage reports quickly.
- Provide reproduction steps for bugs whenever possible.
- Explain the research, dataset, or workflow context behind feature requests.
- Avoid submitting duplicate feedback when an existing item already covers the same issue.
- Do not use feedback, replies, profiles, or reactions for harassment, spam, personal attacks, or misleading claims.

## Good Feedback

High-quality feedback usually includes:

- A concise title
- A clear category
- The expected behavior
- The actual behavior, if reporting a bug
- Why the change matters
- Screenshots or files when they help explain the issue
- Dataset or research context when relevant
- A suggested solution, if available

Examples of useful feedback:

- A reproducible bug in preprocessing or model training
- A feature request that improves student or researcher workflows
- A dataset compatibility issue with a sample file attached
- A performance report with file size, row count, and observed latency
- A UI issue that blocks completion of an existing workflow

## Feedback Categories

Use the most accurate category when submitting feedback.

| Category | Use For |
| :--- | :--- |
| Bug | Broken behavior, errors, failed requests, incorrect output |
| Feature Request | New product capabilities or workflow improvements |
| Dataset | File compatibility, parsing, schema, upload, or dataset handling issues |
| Research | Academic, experiment, report, reproducibility, or citation workflows |
| UI | Usability, accessibility, layout, copy, or interaction issues |
| Performance | Slow processing, timeouts, large-file behavior, or resource usage |
| Other | Valid feedback that does not fit another category |

## Priority Guidance

Priority should describe impact, not personal preference.

| Priority | Meaning |
| :--- | :--- |
| Low | Nice-to-have improvement or minor polish |
| Normal | Useful improvement with moderate impact |
| High | Blocks an important workflow or affects many users |
| Urgent | Prevents core platform usage or causes serious incorrect behavior |

## Reactions

Reactions help administrators understand community sentiment without adding noise.

Supported reactions include:

- Helpful
- Interesting
- Needs More Info
- Agree
- Research Worthy

Use reactions to add signal. Do not use reactions to target or pressure other users.

## Administrator Review

Administrators may review feedback and take the following actions:

- Reply with clarification or decisions
- Assign priority
- Pin important feedback
- Mark duplicate feedback
- Assign ownership
- Add internal notes
- Award administrator stars
- Award badges
- Mark feedback as planned, implemented, closed, or under review

Administrator actions should be consistent, transparent where appropriate, and focused on improving Nexora.

## Administrator Stars

Stars recognize especially useful contributions.

| Stars | Meaning |
| :--- | :--- |
| 1 | Helpful Feedback |
| 2 | Excellent Suggestion |
| 3 | Outstanding Contribution |

Stars contribute to reputation and may appear on profiles, feedback records, and leaderboards.

## Badges and Reputation

Badges and contribution scores are intended to recognize constructive participation. They are not a substitute for respectful conduct or technical quality.

Badges may be awarded for:

- Useful bug reports
- Strong research feedback
- Dataset exploration
- Early testing
- High-impact suggestions
- Consistent community support

Contribution scores may include signals such as submitted feedback, implemented suggestions, administrator replies, badges, stars, and other platform activity.

## Public Profiles and Showcase Content

Users may display community contributions, badges, recent feedback, implemented suggestions, and selected showcase items on their profiles.

Profile showcase fields should remain professional and relevant to Nexora, including:

- Best dataset
- Best research
- Best feedback
- Favorite model
- Pinned achievement

Do not use profile content to impersonate others, publish private information, or misrepresent credentials.

## Attachments

Attachments should support the feedback item.

Allowed uses include:

- Screenshots of UI issues
- Sample files that reproduce dataset problems
- Logs or exported reports relevant to the submission
- Research artifacts that clarify a request

Do not upload sensitive personal data, credentials, private datasets, malicious files, or copyrighted material you do not have permission to share.

## Moderation

Nexora administrators may remove, close, or de-prioritize feedback that is abusive, spammy, irrelevant, duplicative, unsafe, or not actionable. Repeated misuse may result in reduced visibility, loss of recognition, or account restrictions.

## Reporting Issues

If you see harmful behavior, inappropriate content, or a security concern, report it through the appropriate project or repository channel. Security-sensitive issues should not be posted publicly in community feedback.

## Guiding Principle

The best contributions make Nexora easier to test, easier to trust, easier to research with, and easier to improve for everyone.
